# Installing AgentHub v2 (for humans)

Gets you from "nothing" to "logged into a running AgentHub" in about 10 minutes.

## Requirements

- Supported Linux host: Debian/Ubuntu, Fedora/RHEL/Rocky/Alma, Arch, Alpine
- Internet access + sudo (the installer auto-provisions missing prereqs)
- **Ports 80, 443, and 8443 free** (Traefik binds :80 and :443 for AgentHub, :8443 for the Infisical admin console)
- **4 GB RAM, 20 GB disk** minimum

The installer auto-installs what's missing, with your consent, for each of: git, Docker, Docker Compose plugin, Node 22+, pnpm. If everything is already present it skips straight to the TUI.

### Domain (optional)

`localhost` works and gives you a local-only install over HTTP. For a real hostname, point an A record at the host's IP *before* installing so Let's Encrypt's ACME challenge can complete on first boot.

## Install

### Recommended: one-liner

```bash
curl -fsSL https://raw.githubusercontent.com/physikal/agenthubv2/main/scripts/quick-install.sh | bash
```

This asks you (once, per missing prereq) before installing anything. Prefer to read the script first? Good — it's ~360 lines of readable bash:

```bash
curl -fsSL https://raw.githubusercontent.com/physikal/agenthubv2/main/scripts/quick-install.sh -o quick-install.sh
less quick-install.sh
bash quick-install.sh
```

Idempotent — safe to re-run to pull updates.

### Fully unattended

Skip the confirmation prompts (useful in automation / CI / fresh VMs you control):

```bash
curl -fsSL https://raw.githubusercontent.com/physikal/agenthubv2/main/scripts/quick-install.sh \
  | AGENTHUB_AUTO_INSTALL=true bash
```

### Manual (clone-first)

```bash
git clone https://github.com/physikal/agenthubv2.git
cd agenthubv2
./scripts/install.sh
```

Either path runs the same three steps:
1. `pnpm install` — fetches installer deps
2. Builds the two Docker images (server + workspace) locally
3. Runs the Ink TUI installer

See [installer-flow.md](installer-flow.md) for the full step-by-step map of the TUI.

The TUI walks you through:

1. **Prerequisite check.** Verifies Docker, Compose, and ports.
2. **Provisioner choice.** Pick one:
   - **Local Docker** (default) — workspace containers run on the same host. Simplest.
   - **Bundled Dokploy** — spins up Dokploy alongside AgentHub. Use this if you want Dokploy's UI for workspace orchestration.
   - **Remote Dokploy** — point AgentHub at an existing Dokploy instance. Needs its URL + API token + project ID + environment ID.
3. **Domain.** `localhost` or a real hostname.
4. **TLS email.** Only asked for non-localhost domains. Used by Let's Encrypt for cert expiry notices.
5. **Admin password.** Leave blank to auto-generate one (printed at the end).
6. **Confirm → install.** The installer writes `compose/.env`, pulls images, `docker compose up -d`, bootstraps Infisical, force-recreates the server so it picks up the Infisical creds, and prints your credentials.

## When it's done

You'll see two credential sets — **save both**:

```
AgentHub is up at http://localhost
  Admin user:     admin
  Admin password: <your-or-generated>

Infisical console: https://<host>:8443/
  Admin email:    admin@agenthub.local
  Admin password: <auto-generated>

Save these credentials — they are also written to .env.
```

- **AgentHub admin** — log into the web console at the AgentHub URL
- **Infisical admin** — log into the secret-store UI at `https://<your-host>:8443/` (accept the self-signed cert once). Also linked from the Secrets page inside AgentHub.

Both are also in `compose/.env` (look for `AGENTHUB_ADMIN_PASSWORD` and the install stdout — Infisical's is currently only captured in the stdout, so copy it now).

## Verify the install works

Run these from the install host:

```bash
# Health
curl -sf http://localhost/api/health
# → {"status":"ok"}

# All services healthy
docker compose -f compose/docker-compose.yml ps

# Log in
curl -sf -c /tmp/ah-cookie -X POST http://localhost/api/auth/login \
  -H 'Content-Type: application/json' \
  -H 'Origin: http://localhost:3000' \
  -d "{\"username\":\"admin\",\"password\":\"$(grep AGENTHUB_ADMIN_PASSWORD compose/.env | cut -d= -f2)\"}"
# → {"id": "…", "username": "admin", "role": "admin", …}
```

## Your first session

1. Browse to the URL printed at the end (e.g. `http://localhost/`)
2. Log in as `admin`
3. **Settings → Change Password** (immediately)
4. **Sessions → + New session** → name it → create
5. Wait ~10 seconds for status to reach `active`
6. Click the session — you land in an xterm.js terminal inside the workspace container
7. `claude` to launch Claude Code. It'll prompt you for a login on first run.

## Post-install configuration

All of this lives in the web UI under the sidebar. None are required — the install works without any.

- **Integrations** — one page for all external services. Add Cloudflare DNS (apiToken + zoneId), DigitalOcean (apiToken + region + ssh key), Docker host (hostIp + sshPrivateKey), Dokploy (baseUrl + apiToken + project/env), or Backblaze B2 (keyId + appKey + bucket). Each secret lands in Infisical; metadata stays in SQLite. The `agentdeploy` MCP inside workspaces picks these up when the agent invokes `deploy`.
- **Backups** — snapshot `/home/coder` to B2. Save / Restore buttons + run history. Credentials live on the Integrations page (add Backblaze B2 there); this page is operations-only. Backups require an active session.
- **Secrets** — link to the bundled Infisical admin console for folders, environments, secret versions, and audit log. Runs on `https://<your-host>:8443/` (self-signed cert — accept once).

## Upgrading

**Two ways, same result:**

1. **From the web UI** (admin-only) — Settings → **Version** panel. Shows current SHA + latest on `origin/main` + pending commit summaries. Click **Update now** — a one-shot updater container pulls the new code, rebuilds changed images, recreates the stack, and the UI reloads when the new server is healthy. No shell required.

2. **From the host shell**:
   ```bash
   agenthub update          # pull + rebuild + recreate + health probe
   agenthub update --check  # dry-run: show pending commits
   agenthub status          # current SHA + compose services
   ```
   Same code path as the web UI button. `agenthub` was installed to `/usr/local/bin/` by the installer.

Both paths are idempotent. Both honor the existing `compose/.env` and auto-migrate new required env vars when the schema grows. The CLI self-updates as part of the flow, so fixes to `agenthub` itself also land without operator intervention.

## Uninstalling

```bash
cd compose
docker compose down -v                # stops and drops compose volumes
# Workspace volumes persist across session ends by design; purge them:
docker volume ls -q | grep agenthub-home- | xargs -r docker volume rm
```

`.env` file remains; delete by hand.

## Troubleshooting

See [troubleshooting.md](../troubleshooting.md) for the full reference. Common hits:

- **Port 80 / 443 in use** — another web server is running. `sudo ss -lntp | grep -E ':80 |:443 '`.
- **"Secret store not configured" error in UI** — the Infisical bootstrap didn't complete. Re-run `./scripts/install.sh`; it resumes from the right step.
- **Session stuck in "Connecting to agent..."** — workspace container crashed. `docker logs $(docker ps -a --format '{{.Names}}' | grep agenthub-ws- | head -1)`. Most common cause: image build partially failed; rebuild with `docker build --no-cache -f docker/Dockerfile.agent-workspace -t agenthubv2-workspace:local .`.
