# TLS modes

AgentHub supports four TLS strategies. Pick based on whether your install is publicly reachable on `:443` and whether you can give it a DNS provider's API token.

| Mode | When to use | Requires |
|---|---|---|
| `auto` (default) | Most installs ÔÇö let the installer decide | nothing |
| `public-alpn` | Host is on the public internet, port 443 reachable | `AGENTHUB_TLS_EMAIL` |
| `dns-01` | Internal-only, you have a DNS provider API token | `AGENTHUB_TLS_EMAIL`, `AGENTHUB_TLS_DNS_PROVIDER`, provider token |
| `self-ca` | Fully internal, no DNS API, no internet | nothing ÔÇö but each device imports the CA once |

## auto

Inference rules:

- `DOMAIN=localhost` Ôćĺ no cert resolver; Traefik's default cert is served.
- `AGENTHUB_TLS_DNS_PROVIDER` is set Ôćĺ `dns-01`
- otherwise Ôćĺ `public-alpn`

## public-alpn

Let's Encrypt via TLS-ALPN-01. Traefik solves the challenge by accepting an inbound :443 connection from LE's servers and presenting a special cert.

**Required:** `AGENTHUB_TLS_EMAIL=ops@example.com`

**Common failures:**
- ISP blocks inbound :443 Ôćĺ use `dns-01` or `self-ca` instead
- DNS A record missing or wrong Ôćĺ check `dig <domain>`
- Multiple boxes share the same hostname Ôćĺ pick one to certify

## dns-01

Let's Encrypt via DNS-01. Traefik (via lego) provisions a TXT record on your DNS zone, LE verifies it, you get a cert. Works on internal-only hosts.

**Required:**
- `AGENTHUB_TLS_EMAIL`
- `AGENTHUB_TLS_DNS_PROVIDER` ÔÇö lego provider name (`cloudflare`, `route53`, `digitalocean`, `hetzner`, `gandi`, `linode`, ÔÇŽ)
- Provider-specific token:
  - Cloudflare: `AGENTHUB_CLOUDFLARE_API_TOKEN` (or `CF_DNS_API_TOKEN`)
  - Others: pre-export the lego-native env vars (see lego docs)

A pre-flight check validates the Cloudflare token + zone access before flipping config. Skip with `AGENTHUB_SKIP_PREFLIGHT=1`.

**Common failures:**
- Wrong API token Ôćĺ pre-flight catches this for Cloudflare
- Token lacks the right zone Ôćĺ likewise
- Propagation timeout Ôćĺ re-run `agenthub reconfigure-tls` (Plan 4)

## self-ca

A private CA generated on the host. Issues a leaf cert for `<domain>`, `*.<domain>`, and your LAN IP. Each device that connects to AgentHub imports the CA once via `http://<domain>/install/ca`.

**Required:** nothing beyond `AGENTHUB_TLS_MODE=self-ca`. Optionally:
- `AGENTHUB_LAN_IP=ÔÇŽ` ÔÇö override auto-detected LAN IP (comma-separated list for multi-interface hosts)

The CA root is valid for 10 years. The leaf is valid for 27 months and auto-renewed by a sidecar container when it has < 30 days remaining ÔÇö operators don't need to touch it.

### CA distribution

The CA cert is served on **port 80** at:

- `http://<domain>/.well-known/agenthub-ca.crt` ÔÇö direct download
- `http://<domain>/install/ca` ÔÇö platform-aware install instructions (macOS / iOS / Android / Linux / Windows)

These two paths are the only HTTP endpoints not redirected to HTTPS ÔÇö necessary because the browser doesn't trust the CA *until* you import it, so the trust path itself must be plain HTTP. All other `:80` traffic still redirects to `:443`.

### How it works under the hood

- `traefik-self-ca-init` (one-shot init container): generates the CA root + leaf cert via openssl. Idempotent: re-running is a no-op unless `REGEN=1`.
- `traefik-self-ca-renew` (long-running sidecar): daily check; regenerates the leaf if `< 30 days` remaining. CA root is left alone.
- `agenthub-static` (nginx sidecar): serves the CA cert + install page on port 80 via Traefik routes.

All three live in `compose/traefik.override.yml` (generated by the installer) and use a shared `traefik-self-ca` Docker volume for the cert files.
